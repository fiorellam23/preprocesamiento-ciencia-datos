# DOCUMENTACIÓN DEL PROYECTO

## Introducción
Este proyecto tiene como objetivo aplicar técnicas de preprocesamiento de datos utilizando Python y la librería Pandas, junto con el uso de Git y GitHub para el control de versiones.

Se desarrollaron funcionalidades básicas para limpiar y preparar datos, permitiendo mejorar la calidad de la información antes de su análisis.

---

## Funcionalidades
Se implementaron funciones para:
- Cargar datos
- Eliminar valores nulos
- Eliminar duplicados
- Normalizar datos
- Codificar variables categóricas

---

## Comandos Git utilizados

- git clone: Permite clonar un repositorio desde GitHub a la computadora local.
- git add: Agrega archivos al área de preparación para ser guardados.
- git commit: Guarda los cambios realizados en el proyecto.
- git push: Envía los cambios al repositorio en GitHub.
- git branch: Permite crear nuevas ramas de trabajo.
- git checkout: Permite cambiar entre ramas o versiones del proyecto.
- git merge: Une los cambios de una rama con otra.

---

## Flujo de trabajo
1. Se creó el repositorio en GitHub.
2. Se clonó el repositorio en el entorno local.
3. Se creó una rama llamada feature-preprocesamiento.
4. Se desarrolló el archivo preprocesamiento.py.
5. Se realizaron commits y se subieron los cambios.
6. Se creó un Pull Request en GitHub.
7. Se realizó la fusión (merge) con la rama principal.

---

## Automatización
Se implementó GitHub Actions como herramienta de automatización. 
Esta permite ejecutar procesos automáticos dentro del repositorio, 
como validación de código o pruebas.

Aunque no se configuró un workflow avanzado, se comprende que GitHub 
Actions permite mejorar la calidad del desarrollo mediante integración continua.
